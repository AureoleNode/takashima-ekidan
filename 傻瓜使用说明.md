# 高岛断易 · 傻瓜使用说明

> 不懂《易经》也可以使用。这个工具只负责生成和保存卦象，不会替你解释吉凶，也不会替你做决定。

## 1 分钟上手

1. 打开[在线版](https://aureolenode.github.io/takashima-ekidan/)，或双击下载目录里的 `index.html`。
2. 在“所占何事”中写一个明确的问题。
3. 点击一次“一键起卦”。
4. 查看本卦、动爻、之卦、互卦和原始数列。
5. 点击“复制卦象”直接粘贴给 AI，或点击“导出卦象”保存文件。

```text
打开工具 → 写一个问题 → 起一卦 → 查看结果 → 复制或导出
```

## 第一步：选择在线版或离线版

### 在线使用

直接打开：

<https://aureolenode.github.io/takashima-ekidan/>

不需要安装，电脑和手机的现代浏览器都可以尝试使用。

### 离线使用

下载整个项目后，双击 `index.html`。页面打开后不需要登录，也不需要联网。

在线版和离线版使用相同的代码。在线版需要先从 GitHub 加载网页，但你输入的问题和生成的卦象仍在浏览器中计算，本工具不会主动上传这些内容。问题非常敏感时，建议使用离线版。

## 第二步：只问一件明确的事

在“所占何事”里，一次只写一个问题。最好同时写明时间范围和判断对象。

推荐：

```text
在未来三个月内，我是否应该继续执行当前的交易方案？
```

不推荐：

```text
我的事业、感情、财运以后会怎么样？
```

第二种写法一次混合了多件事，之后很难判断卦象究竟对应什么。

可以参考这个句式：

```text
在【时间范围】内，我是否应该【具体行动】？
```

问题中不要填写身份证号、银行卡号、账号密码、详细住址或无关的他人隐私。

## 第三步：点击一次“一键起卦”

确认问题后，点击“一键起卦”。页面会模拟六次三钱法，从初爻到上爻自动生成完整卦象。

- 同一个问题只点击一次；
- 成卦后按钮会锁定；
- 导出文件名会使用当天日期；
- 工具不会读取联系人、相册、摄像头或其他个人资料。

不要因为不喜欢结果就立刻重复起卦。如果确实要问一个新问题，可以在结果页点击“返回重起”。

> [!WARNING]
> “返回重起”会清空当前问题和卦象。需要保留时，请先复制或导出。

## 第四步：看懂结果页上的名称

成卦后，页面会切换到结果页：

- **本卦**：六次结果最初形成的卦；
- **动爻**：数值为 6 或 9、会发生变化的爻；
- **之卦**：动爻变化后形成的卦；没有动爻时，本卦不变；
- **互卦**：使用本卦中间四爻重新组合的卦，可作为内部结构参考；
- **原始数列**：六次结果，顺序固定为“初爻到上爻”，也就是自下而上。

数值规则：

| 数值 | 名称 | 状态 |
|---:|---|---|
| 6 | 老阴 | 动爻，阴变阳 |
| 7 | 少阳 | 静爻 |
| 8 | 少阴 | 静爻 |
| 9 | 老阳 | 动爻，阳变阴 |

页面只展示计算结果，不会自动给出吉凶、建议或结论。

## 第五步：复制、保存或重新开始

结果页有三个按钮：

- **返回重起**：清空当前内容，返回提问页面；
- **复制卦象**：把问题与完整卦象复制到剪贴板，可直接粘贴到 AI 对话；
- **导出卦象**：把当前问题和卦象下载为 Markdown 文件。

复制和导出的文字内容完全一致。正常情况下，点击“复制卦象”后会显示“已经复制”；切换到 AI 对话框，使用 `Ctrl + V`（手机端长按后选择“粘贴”）即可。

如果浏览器禁止读取剪贴板，页面会提示复制失败，此时改用“导出卦象”即可。

导出文件通常会进入浏览器设置的下载目录。在 Windows 中一般是“下载”文件夹；手机浏览器的位置因系统而异。

Markdown 正文包含：

- 问题；
- 本卦；
- 动爻；
- 之卦；
- 互卦；
- 自下而上的原始数列。

它不包含分析结论、AI 提示词或自动解读。

## 第六步：整理个人记录（可选）

只想临时使用时，保留下载的 Markdown 即可。

需要长期复盘时，可以在自己的电脑上建立：

```text
卦象/   # 原始问题和原始卦象
解读/   # AI 解读、自己的判断和事后复盘
```

建议文件名：

```text
2026-07-23_未来三个月是否继续当前方案.md
```

不要把事后解释写回原始卦象文件。公开仓库默认忽略 `卦象/` 和 `解读/`，但上传 GitHub 前仍要亲自检查 `git status` 和待提交文件。

## 第七步：需要时再交给 AI

最省事的方法是点击“复制卦象”，然后在自己选择的 AI 对话中粘贴，再补充下面的要求：

```text
请分析这份卦象，逐项说明依据、可能的不同解释和不确定性。
不要把 AI 生成内容冒充为《高岛易断》原文。
```

请注意：本工具不会上传问题，但当你主动把文件发送给 AI、聊天软件或网盘时，内容会交给相应的第三方服务。发送前应删除不必要的姓名、账号、联系方式、住址、证件号码和他人隐私。

原典内容应注明书名、版本和页码。没有可靠出处时，应写成“AI 分析”或“个人理解”，不能写成原文。

## 第八步：事情结束后复盘

在单独的复盘文件中记录：

- 实际发生了什么；
- 哪些判断吻合；
- 哪些判断冲突；
- 当时遗漏了什么信息；
- 是否存在解释过于宽泛、任何结果都能套上的情况；
- 现实证据是否与当时的解释一致。

既记录“准的”，也记录“不准的”。原始问题和卦象保持不变，才能在事后进行核对。

## 常见问题

### 在线使用会上传我的问题吗？

不会。本项目没有提交问题的接口，也没有分析统计或第三方脚本。GitHub 作为在线托管方仍会收到普通网页访问所必需的请求信息，例如 IP 地址、浏览器类型和访问时间。高度敏感的问题建议离线使用。

### 为什么点击“一键起卦”没有反应？

先确认“所占何事”不是空白。如果页面仍无反应，请刷新后重试，或换用较新的 Chrome、Edge、Firefox、Safari 浏览器。

### 点击“返回重起”后还能恢复吗？

不能。返回重起会清空当前问题和结果，所以重要卦象应先复制或导出。

### 点击“复制卦象”后复制了什么？

复制内容与导出的 Markdown 相同，包括问题、本卦、动爻、之卦、互卦和自下而上的原始数列，不包含自动分析或隐藏提示词。

### 为什么复制失败？

部分旧浏览器或严格的剪贴板权限设置可能阻止复制。请确认操作发生在你主动点击按钮之后；仍然失败时，使用“导出卦象”并打开文件复制。

### 为什么找不到导出的文件？

先查看浏览器的下载记录，再检查系统“下载”文件夹。手机浏览器可以检查“文件”“下载项”或浏览器自己的下载管理页面。

### 可以连续问很多问题吗？

可以问不同问题，但每一卦都应对应一个明确问题。不要因为不满意第一次结果而对同一个问题连续重起。

### 结果可以直接当作医疗、法律或交易建议吗？

不可以。卦象属于传统文化与解释性材料，不能替代现实证据、风险控制和专业意见。

## 推荐的固定流程

```text
一个明确问题
→ 写明必要的时间范围
→ 只起一次
→ 先复制或导出原始卦象
→ 需要时再进行解读
→ 事情结束后单独复盘
```
